## What this repo is

A static export of a Next.js 16 site (App Router, static `output: "export"`).
The contents are generated by `next build` and committed here so GitHub Pages
can serve them with no build step and no runtime cost.

## How it got here (publish flow)

The source is built locally and synced into this repo â€” **not** built by GitHub
Actions. Steps, run from the source project:

```bash
# 1. Build the static export for the custom domain (no basePath):
NEXT_PUBLIC_SITE_URL=https://www.badaltutorial.academy npm run build
# â†’ out/

# 2. Sync into a clone of THIS repo, preserving CNAME + .nojekyll + README:
rsync -a --delete \
  --exclude=.git --exclude=.nojekyll --exclude=CNAME --exclude=README.md \
  out/ ./badal-tutorial-website/

# 3. Commit + push main â†’ GitHub Pages auto-deploys.
```

The `--exclude=CNAME`, `--exclude=.nojekyll`, and `--exclude=README.md` are
critical: `rsync --delete` would otherwise remove them and break the custom
domain / asset serving / this README.

## Deployment notes (GitHub Pages custom domain)

- **DNS:** four apex `A` records â†’ GitHub Pages IPs
  (`185.199.108/109/110/111.153`), plus `www` `CNAME` â†’ `<user>.github.io`.
- **Canonical:** `www.badaltutorial.academy` is primary; the apex redirects to
  `www`. GitHub's Let's Encrypt cert covers both.
- **HTTPS:** "Enforce HTTPS" toggled in Pages settings.
